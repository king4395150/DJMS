# 产品需求文档 (PRD)：党群工作机构管理模块 (V2.0 基于Figma原型)

## 1. 需求背景

为落实集团党建工作相关制度要求，支持集团党委动态、精准掌握各业务单元（集团-二级单位两级）的党群工作机构设置及编制配备情况，需在现有党建系统中新增「党群工作机构管理」模块，实现机构编制数据的线上化与动态化管理。

## 2. 业务目标

1. **落实制度**：满足集团对党群工作机构编制管理的规范化、线上化制度要求。
2. **动态掌握**：实现机构编制、专职党务干部、人员名单的数据化展示，确保集团底数清、情况明。
3. **定期核对**：通过润工作机器人每半年推送核对卡片，确保数据准确性和时效性。

## 3. 用户角色

| 角色名称 | 核心职责 |党委专职副书记
| :--- | :--- |
| **集团党建管理员** | 全局监督、数据抽查、查看全量数据及操作日志。 |
| **业务单元党建管理员** | 本单位数据填报、维护、处理润工作定期核对卡片。 |

## 5. 页面入口

- PC端菜单路径：`首页` -> `党组织` -> `党群机构管理`（左侧二级菜单高亮选中态：橙色右边框+浅橙背景）。
- 移动端入口：润工作机器人推送核对卡片（P-008），点击卡片按钮跳转至对应 H5 页面（P-009、P-010、P-011）。

## 6. 核心流程

### 6.1 数据联动流程

| 触发事件 | 动作 | 影响 |
| :--- | :--- | :--- |
| 员工离职（HR系统同步） | 判断员工状态，若为离职则从人员名单移除 | `现有编制人数`减1 |
| 组织关系转出审批通过 | 党员转出且审批通过，从人员名单移除 | `现有编制人数`减1 |
> 离职是最优先判定条件，覆盖所有在岗职工（含非党员）；非离职场景下仅党员的组织关系转出触发名单变动。

### 6.2 定期核对流程（基于润工作机器人）

- 系统**每半年**（6月30日和12月31日早上10:00）自动推送核对卡片至业务单元管理员润工作IM
- 卡片展示该机构完整数据摘要
- 管理员在卡片上可选择三个操作之一：
  - **【确认信息】**：数据无误，确认后卡片翻转为「已确认」状态，展示确认人及确认时间，更新`最近确认时间`
  - **【填写特殊说明】**：跳转至移动端特殊说明填写页（P-010），对有异常的数据填写说明、预计完成时间及说明材料
  - **【修改信息】**：跳转至移动端详情编辑页（P-009），修改机构信息及人员名单

#### 卡片状态流转

| 状态 | 说明 |
| :--- | :--- |
| **待确认** | 初始状态，展示三个操作按钮 |
| **已确认** | 点击【确认信息】后翻转，按钮隐藏，展示确认人+确认时间 |

- 【填写特殊说明】和【修改信息】保存后**仅更新卡片数据摘要**，不改变待确认状态
- 确认后修改数据**无需二次确认**，修改即生效

> 卡片数据为推送时生成的静态快照。已确认卡片后续不会随数据变化自动翻转回待确认状态。

### 6.3 党群部职能设置

- 预设字典维护面板，用于管理`组织建设、政治理论等`的可选值
- 支持增删改：组织建设、政治理论、宣传思想、文化建设、舆情管理、群团统战、信访维稳、社会参与等

## 7. 页面详情

### 7.1 P-001 机构编制列表页

**页面布局：** 组织切换筛选器 + 表格（无新建功能，机构数据来自基础信息库）

**表格列（从Figma组件推断）：**

| 列名 | 说明 | 数据来源 |
| :--- | :--- | :--- |
| 党组织名称 | 组织名称，支持点击穿透至详情页 | 基础信息库 |
| 关联对应部门 | 本党组织下的党群工作机构是哪个部门，在编辑页手动关联 |
| 是否成立党群部门 | 是或否，在编辑页手动选择 |
| 计划编制人数 | 规划的编制数 | 详情页手动维护 |
| 现有编制人数 | 人员名单内有效人数 | 自动统计 |
| 党委专职副书记 | 副书记姓名 | 来自党员信息模块党内职务，只读展示 |
| 更新人 | 最近一次数据更新人 | 编辑信息或点击确认都算 |
| 更新时间 | 最近一次数据更新时间 | 编辑信息或点击确认都算 |
| 特殊说明 | 异常情况备注 | 业务单元在特殊说明页填写 |
| 操作 | [特殊说明]按钮 | - |

**筛选区：** 组织切换下拉框，集团管理员可查看全量，业务单元管理员仅看本级数据。

#### 区域1 机构信息编辑

| 字段 | 控件类型 | 说明 |

| 计划编制人数 | 数字输入框（可编辑） | 整数 ≥0 |

表格列，除查看模式所有列外，新增操作列：

| 列名 | 说明 |
| :--- | :--- |
| 是否为管理团队成员 | 可编辑下拉：正职/副职/不涉及/请选择 |

2. **排序**：支持各数值列（计划编制人数、现有编制人数、差额、党员占比、35岁及以下占比等）点击表头排序
3. **职能列展开**：因职能列较多（8个职能各占一列），默认在视口中显示前6个职能列，其余在横向滚动中查看；
4. **行点击**：点击任意行的组织名称跳转至该机构详情页（P-002 查看模式）

## 10. 权限规则

- **数据权限**：集团党建管理员可查看全量数据；业务单元党建管理员仅可查看本单位数据
- **操作权限**：集团和业务单元党建管理员可编辑计划编制人数、是否成立党群部门等字段
- **日志权限**：仅集团党建管理员及系统管理员可查看全量操作日志，在数据库查看，暂时不做页面
-  **人员选择器、组织选择器权限**：根据当前用户有权限管理的党组织对应的管理架构，下的所有管理架构节点和人员

## 11. 校验规则

1. `计划编制人数`：必须为 ≥0 的整数
2. `预计完成时间`：非强制必填，异常状态下可仅填写特殊说明进行报备
3. `党委专职副书记`：只读字段，数据来源为党员信息模块党内职务，不在此模块选择或修改
4. `职能`：必须使用字典值（P-006）下拉/标签选择，禁止手动输入
5. `特殊说明`：500字以内
6. `附件`：仅支持 .doc/.docx/.pdf，单文件 ≤30M，最多5个附件

## 12. 异常场景

1. **离职流程卡点**：员工已离职但审批卡点未通过，人员名单未消除。处理方案：系统不自动干预，需管理员线下催促审批或手动发起转出流程。
2. **润工作回调丢失**：用户点击【确认信息】后网络波动导致系统未收到回调。处理方案：前端防抖处理，超时后Toast提示"网络异常，请重试"，不更新本地UI状态。
3. **附件上传失败**：用户填写特殊说明时附件上传中断。处理方案：前端提示重试，已填写的说明文本不丢失。

## 15. 测试点

1. **UI与交互测试**：列表页异常状态标红；润工作卡片三按钮交互；职能设置面板增删改；附件上传限制

4. **边界值测试**：计划编制输入负数/小数/超大整数；特殊说明超500字；附件超30M/超5个/格式不对

6. **日志审计测试**：修改编制、点击确认后，操作日志记录变更前后快照及操作人信息
